# Slides

One PDF per lecture, numbered in teaching order.

## Naming convention

```
lecture-01-introduction.pdf
lecture-02-data-wrangling.pdf
lecture-03-descriptors.pdf
...
```

- **Lowercase**, hyphen-separated, no spaces.
- **Zero-padded** (`lecture-01`, not `lecture-1`) so `lecture-10` sorts after
  `lecture-02`, not between `lecture-1` and `lecture-2`.
- **ASCII only** in filenames — no *æ, ø, å* — because Bash and Python
  sometimes stumble on them. Inside the file, write in any language you like.

## Why PDF?

PDF is the universal format: it opens in every browser, on every OS, without
proprietary software. Keep your editable source (`.pptx`, `.key`, `.odp`) if
you want — push them to `slides/sources/` if you do — but the PDF is the
canonical version students read.

## FAIR role

**Accessible** (opens anywhere) and **Interoperable** (open format).
