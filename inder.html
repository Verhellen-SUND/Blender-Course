Blender basics

Before working with molecular data, you need to be comfortable with the Blender interface: navigating the viewport, transforming objects, and understanding how lighting and render engines affect the final output. These four cards cover the concepts and shortcuts you will return to throughout the day.

Render & lighting
Render engines & lighting setup
Blender has two main render engines with very different speed–quality trade-offs. Lighting has an equally large impact on the final image and is straightforward to set up correctly from the start.
Expand ↓
Render engine choice
EeveeReal-time rasterisation. Fast, works on any hardware. Suitable for previews and presentations.
CyclesPath-traced, physically accurate. Much slower. Use for final publication-quality images.
Basic lighting setup
  1. Delete the default point light. Add a Sun lamp: Shift + A → Light → Sun.
  2. Rotate it to come from roughly 45° above the scene.
  3. In World Properties, set the background strength to 0 for a black background.
  4. Optionally add a second, weaker light on the opposite side to soften shadows.
Material quick settings
  • Increase Roughness for a matte, non-reflective surface
  • Add a small amount of Emission for a subtle internal glow effect
  • Use Subdivision Surface on ribbon geometry to smooth curves
Geometry nodes

Geometry nodes is Blender's procedural modelling system. You connect nodes in a graph to describe how geometry is constructed and transformed. Understanding this before loading molecular data means you will recognise what Molecular Nodes is doing under the hood, rather than treating it as a black box.

Core concept
What is a node graph?
A node graph is a visual program. Each node is a function that takes inputs on the left and produces outputs on the right. You connect them with wires to build a processing pipeline.
Expand ↓
Key terms
  • Node — a single operation (e.g. "convert mesh to points")
  • Socket — the coloured dot on a node where connections attach
  • Link — the connection (wire) between two sockets
  • Group Input / Output — the entry and exit points of the graph
Node editor shortcuts
Shift + AAdd a new node (opens search menu)
Ctrl + XDelete node, preserve connected links
XDelete selected node and its links
Ctrl + Shift + ClickPreview a node's output in the viewport
MMute (temporarily disable) a node
Core recipe
Points → instances
The most important pattern in Geometry Nodes — and exactly what Molecular Nodes uses for atoms. Convert geometry to points, then place a shape on each point.
Expand ↓
Step-by-step
  1. Add a Cube. Open the Geometry Nodes workspace. Click New.
  2. Add a Mesh to Points node — the cube becomes 8 points.
  3. Add an Instance on Points node. Connect Points → Points socket.
  4. Add an Icosphere node. Connect its Geometry output → Instance socket.
  5. Adjust the Icosphere radius to control sphere size.
This is precisely what Molecular Nodes does: PDB atom coordinates become points, and a sphere is instanced on each one. The generated graph will look familiar after this exercise.
Extension — add displacement
  1. Add a Set Position node before Instance on Points.
  2. Add a Noise Texture node. Connect its Color output → Offset input of Set Position.
  3. Increase Scale and Strength on the Noise Texture to spread and amplify the displacement.

These nodes appear frequently in both general Geometry Nodes work and in the graphs that Molecular Nodes generates automatically. Recognising them saves time when reading the generated graph.

Molecular nodes

Molecular Nodes is a Blender add-on that imports structural biology data — PDB structures, mmCIF files, and MD trajectories — and builds a Geometry Node graph automatically. Because you have already worked with Geometry Nodes directly, the generated graph will be readable rather than opaque.

Setup
Installing Molecular Nodes
The add-on is installed once through Blender preferences and then accessed through a dedicated panel in the 3D viewport side panel.
Expand ↓
Installation steps
  1. Go to Edit → Preferences → Add-ons.
  2. Click Install and select the downloaded .zip file.
  3. Enable the add-on by ticking the checkbox next to its name.
  4. In the 3D viewport, press N to open the side panel. A Molecular Nodes tab will appear.
Accessing the panel
  • The panel is docked in the N panel of the 3D Viewport, not in a separate window
  • All import options, representation presets, and trajectory controls are found here
Importing
Loading a PDB structure
You can fetch structures directly from the RCSB by PDB ID, or load a local file. The add-on parses the atom data and builds the node graph automatically.
Expand ↓
Steps
  1. In the Molecular Nodes panel, type a PDB ID (e.g. 1CRN) in the fetch field.
  2. Click Download. Blender fetches the structure from RCSB and imports it.
  3. A new object appears in the viewport and a Geometry Node graph is created automatically.
  4. Switch to the Geometry Nodes workspace to inspect the generated graph.
Recommended structures to start with
  • 1CRN — crambin, very small (46 residues), loads and renders quickly
  • 1BNA — B-form DNA double helix, immediately recognisable
  • 6VXX — SARS-CoV-2 spike protein, visually striking but more demanding on hardware
Representations
Changing the visual style
Molecular Nodes provides preset node groups for the standard structural representations used in publications. These can be combined within the same scene.
Expand ↓
Standard representations
Ball and StickSpheres on each atom with bond cylinders. Best for small molecules and active site detail.
Ribbon / CartoonSecondary structure as ribbons, arrows, and tubes. Standard for protein overviews.
SurfaceSolvent-accessible surface. Useful for binding pockets and electrostatic figures.
SticksBond geometry without atom spheres. Common for ligands in active site figures.
Colouring by attribute
  • Use Color by Element for standard CPK colouring
  • Use Color by Chain to distinguish protein subunits
  • Use Color by Secondary Structure with the ribbon representation
  • Use a Color Ramp on B-factor or hydrophobicity for research figures
Data model
How PDB data maps to nodes
Understanding the data model prevents confusion when navigating the node graph. Each atom is a point; its chemical properties are stored as named attributes on that point.
Expand ↓
Data structure
PointOne atom. Its position = XYZ coordinates from the PDB ATOM record.
EdgeOne covalent bond between two atoms.
AttributePer-point data: atomic number, chain ID, residue index, residue name, B-factor, and more.
Reading an attribute in the graph
  1. Add a Named Attribute node (Shift + A → Attribute → Named Attribute).
  2. Type the attribute name, e.g. atomic_number, chain_id, or b_factor.
  3. Connect the output to a Compare node to filter atoms, or to a Color Ramp to map values to colour.

MD trajectories
Importing a trajectory
Molecular Nodes supports XTC and multi-model PDB trajectory files. Each simulation frame maps to a Blender frame. A smoothing node reduces the high-frequency positional noise common in MD data.
Expand ↓
Steps
  1. In the Molecular Nodes panel, switch to the Trajectory tab.
  2. Load your topology file (e.g. .gro or .pdb) and your trajectory file (.xtc).
  3. Press Spacebar to play. The frame counter at the bottom shows the current frame.
  4. If playback is slow, reduce the viewport subdivision level or switch to Eevee in Render Properties.
Smoothing the trajectory
  • Add an Interpolate node (or MD Smoothing) to the node graph
  • Increase the Window value to average over more frames — this removes high-frequency jitter without distorting the overall motion
Keyframes
Camera & object animation
A slowly rotating protein is one of the most effective visualisations for a presentation. It requires only two keyframes on the object's Z rotation.
Expand ↓
Setting up a rotation loop
  1. Go to Frame 1 (type 1 in the frame counter at the bottom).
  2. Select the protein object. Press I → choose Rotation to insert a keyframe.
  3. Go to Frame 250 (or your chosen end frame).
  4. Press R → Z → 360 to rotate 360°. Press I → Rotation again.
  5. The protein now completes one full rotation over 250 frames (at 24 fps ≈ 10 seconds).

Exporting
Rendering & saving output
Blender can output a single still image or a full animation sequence. For animations, always render to a numbered image sequence rather than directly to video.
Expand ↓
Render shortcuts
F12Render current frame (still image)
Ctrl + F12Render animation (all frames in sequence)
F11Open the last rendered image
Saving a still image
  1. After rendering with F12, the result opens in the Render Result window.
  2. Go to Image → Save As.
  3. Choose PNG (lossless, suitable for publications) or JPEG for quick sharing.
Saving an animation
  1. In Output Properties, set the output folder and format to PNG.
  2. Render with Ctrl + F12. Each frame saves as a numbered PNG file.
  3. Combine to video using the Blender Video Sequencer or FFmpeg.
Rendering to a PNG sequence rather than MP4 directly means that if Blender closes mid-render, all completed frames are preserved on disk.
Quality
Improving output quality
Several settings have a large impact on render quality with relatively modest cost. These are particularly relevant when producing images for publications or conference posters.
Expand ↓
Cycles — reducing noise
  • Increase Samples (Render Properties → Sampling) — 128 for drafts, 512+ for final output
  • Enable Denoising in Render Properties: use OptiX on NVIDIA GPUs or OpenImageDenoise on any hardware. This dramatically reduces visible grain at lower sample counts.
  • Enable Denoising Data in View Layer Properties if you prefer to denoise during post-processing
Resolution & file format
  • For journal figures, 300 DPI at the intended print size is standard — for a 90 mm wide figure that is approximately 1063 × 1063 px
  • Use PNG 16-bit for maximum colour depth when the image will be post-processed in Photoshop or Illustrator
  • Use OpenEXR if you need to adjust exposure or re-composite lighting after the render is complete
Colour management
  • In Render Properties → Color Management, set View Transform to AgX (Blender 4.0+) or Filmic for natural, publication-appropriate tones
  • Adjust Exposure and Gamma here rather than in the material settings for consistent, predictable results
